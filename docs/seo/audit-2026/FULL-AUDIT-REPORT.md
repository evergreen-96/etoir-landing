## 🔴 КРИТИЧЕСКИЕ ФИКСЫ — обязательно до деплоя

### К1. `nginx/etoir.conf`: подтверждённый soft-404

В конфиге:

```nginx
location / {
    ...
    try_files $uri $uri/ /index.html;   ← soft-404
}
```

Любой несуществующий URL → отдаётся главная с `200`. Поисковики увидят «уникальный URL → контент главной → canonical=`/`» → склейка + потерянный crawl-бюджет.

**Действие — два изменения:**

1. В блоке `location /` поменять последнюю строку `try_files`:

   ```nginx
   try_files $uri $uri/ $uri/index.html =404;
   ```

2. Добавить кастомную 404-страницу:

   ```nginx
   error_page 404 /404.html;
   location = /404.html { internal; root /usr/share/nginx/html; }
   ```

3. Создать `landing/404.html` (минимум: тот же header/footer что у блога, заголовок «404 — страница не найдена», ссылка на главную и поиск по блогу).

### К2. Все три блог-статьи: НОЛЬ реальных картинок

В моём предварительном замере на блог-страницах было «1 картинка». При глубокой проверке выяснилось: это **трекинг-пиксель Яндекс.Метрики** `https://mc.yandex.ru/watch/107942912`. Фактических иллюстраций — **ноль**.

При этом:
- В подпапках `landing/blog/<slug>/` нет каталога `img/` или `images/` — структуры под картинки тоже нет.
- `og:image` у всех 3 статей = общая `https://etoir.ru/images/og-image.png` — нет уникальной обложки на статью.

**Последствия:** статья 5–8 тыс. знаков сплошного текста без визуала — высокий показатель отказов (40–55 % типично), просадка по поведенческим, нулевая позиция в Яндекс.Картинках.

**Действие на каждую статью:**
- Создать подпапку `landing/blog/<slug>/img/`.
- Подготовить минимум 4 картинки: hero (1200×630, ≤ 200 КБ) + 2–3 иллюстрации/схемы/скриншота по тексту.
- WebP + PNG fallback через `<picture>`.
- alt с осмысленным русским ключом.
- Уникальный `og:image` (hero статьи).
- `<link rel="preload" as="image" type="image/webp" href="/blog/<slug>/img/hero.webp">` в `<head>`.

### К3. Все три блог-статьи: НОЛЬ внутренних ссылок (битая перелинковка)

В `cmms-vs-excel`: 0 ссылок на главную, 0 ссылок на другие статьи. То же в `kak-vnedrit` и `zachem`.

При этом видны 3–4 якоря `#contact-form` (CTA «Получить демо»). **Эти якоря битые:** форма находится на главной, а `#contact-form` без слеша интерпретируется браузером как «якорь текущей страницы» → CTA-кнопки никуда не ведут.

**Действие:** в каждой блог-статье заменить все `#contact-form` на `/#contact-form` (а лучше — на абсолютный `https://etoir.ru/#contact-form`). Дополнительно добавить:
- ≥ 3 ссылки на главную (включая `/#features-section`, `/#industries`, `/#roi-calculator`)
- ≥ 1 ссылку на каждую другую блог-статью (формирует кластер блога)

### К4. Блог-статьи: `<meta name="robots">` MISSING у 2 из 3

| Статья | robots meta |
|---|---|
| `cmms-vs-excel` | **MISSING** |
| `kak-vnedrit-toir-sistemu` | **MISSING** |
| `zachem-avtomatizirovat-toir` | `index, follow` ✓ |

**Действие:** проставить во всех:

```html
<meta name="robots" content="index, follow, max-image-preview:large">
```

`max-image-preview:large` обязательно — Яндекс.Картинки и AI Overviews обрезают превью по умолчанию.

### К5. `zachem-avtomatizirovat-toir`: H2 без `id` — нет якорей

В этой статье 6 H2 (6 причин), у всех `id` отсутствует. Это значит:
- нет оглавления с быстрыми переходами
- Яндекс не подхватит «быстрые ссылки» в сниппет

**Действие:** проставить `id` на каждый H2: `#reason-1`, `#reason-2`, ..., `#reason-6` и добавить блок «Содержание» (как в других статьях).

## 🟠 ВЫСОКИЙ ПРИОРИТЕТ — желательно до деплоя

### В1. `<meta name="twitter:card">` MISSING везде (главная и все 3 блог-статьи)

Open Graph есть, Twitter Card — нет. Большие превью в X/Twitter и Telegram опираются на `twitter:card`.

**Действие:** в `<head>` всех страниц:

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="<совпадает с og:title>">
<meta name="twitter:description" content="<совпадает с og:description>">
<meta name="twitter:image" content="<совпадает с og:image>">
```

### В2. Главная: `<img>` с `src="undefined"`

При парсинге `<img>`-тегов на главной найден элемент `#10` с `src=undefined, alt=MISSING`. Это либо JS-инжектированный тег с не успевшим инициализироваться `src`, либо опечатка в шаблоне.

**Действие:** грепнуть `landing/index.html` по `src="undefined"` или `src='undefined'`, найти и починить либо удалить.

### В3. Главная: preload — относительный путь `images/top-dash.webp`

Сейчас в `<head>`: `<link rel="preload" as="image" href="images/top-dash.webp">`. Безопаснее абсолютный путь:

```html
<link rel="preload" as="image"
      href="/images/top-dash.webp"
      type="image/webp"
      fetchpriority="high">
```

### В4. Главная: GA активен (`gtag.js?id=G-DS3D1SLNHD`) — в РФ малополезен

С 2022 GA-домены в РФ периодически блокируются провайдерами. У вас уже подключена Яндекс.Метрика — она основная.

**Действие:** временно убрать GA до подтверждения нужности по статистике YM (доля зарубежного трафика).

### В5. Главная: 0 внешних ссылок — слабый E-E-A-T

Internal links: 29. External: **0**. Для B2B-SaaS, особенно при ставке на «российская система из Реестра Минцифры», отсутствие исходящих ссылок на trust-источники — негативный сигнал.

**Действие:** добавить 3–5 ссылок:
- профиль в Реестре отечественного ПО (Минцифры) — когда попадёт
- ISO 14224 / ГОСТ 18322 (стандарты надёжности и ТОиР)
- 1–2 кейса клиентов (их сайты)
- профиль на Хабре / vc.ru / РБК (если есть публикации)

### В6. Главная: 2 `<img>` без `alt` из 11 (исключая пиксель Метрики)

Уже идентифицированы: `#0` (пиксель — норм) и `#10` (битый src — фикс В2). Если останутся декоративные иконки — поставить `alt=""` явно.

### В7. Главная: 2 картинки после первой без `loading="lazy"`

В аудите видно: `#1` (top-dash) — eager, корректно (above the fold). Остальные 8 — lazy. Но в подсчёте 2 пропущенных — нужно перепроверить, какие именно.

**Действие:** грепнуть `<img` в `landing/index.html` и проставить `loading="lazy"` всем кроме первой.

### В8. Блог: длина статей в 1.5–2× меньше нашего ТЗ-стандарта

| Статья | Видимый текст | По ТЗ норма |
|---|---:|---:|
| `cmms-vs-excel` | 7 427 знаков (1 042 слова) | 10 000–12 000 |
| `kak-vnedrit-toir-sistemu` | 7 572 знаков (1 026 слов) | 14 000–16 000 |
| `zachem-avtomatizirovat-toir` | 5 238 знаков (714 слов) | 8 000–10 000 |

**Решение:** не дотягивать «вручную» — после деплоя планомерно расширять каждую статью при следующем `dateModified`-обновлении: добавлять FAQ, расширять разделы, вставлять картинки и таблицы. См. обновлённое ТЗ в `docs/specs/blog-seo-tz-2026.md`, раздел «Апгрейд существующих статей».

### В9. rss.xml: Yandex Turbo namespace объявлен, но не используется

`xmlns:turbo="http://turbo.yandex.ru"` в `<rss>` есть, но `<turbo:content>` в `<item>`-ах отсутствует.

**Действие — выбрать:**
- Если Yandex.Turbo нужны (быстрые мобильные страницы) — добавить `<turbo:content>` в каждый item с упрощённым HTML.
- Если не нужны — убрать namespace (Я.Турбо постепенно сворачивается в пользу обычной мобильной выдачи).

Рекомендую **убрать** — Турбо сейчас даёт всё меньше преимуществ, добавляет работу.

### В10. Блог: уникальный `og:image` на каждую статью

Сейчас у всех `og:image = /images/og-image.png` (общая). После фикса К2 каждая статья получит свой hero — его и подставлять в og:image, twitter:image, RSS enclosure.

```nginx
location ~* \.(docx|xlsx|pdf)$ {
    add_header X-Robots-Tag "noindex, follow" always;
    add_header Content-Disposition 'attachment';
    expires 30d;
}
```
