**Вывод:** кластер высокочастотный, но контентно пустой. Запрос «график ППР … образец/скачать» имеет **документ-интент** (человек хочет готовый файл) + **how-to интент** (как составить). Ни один конкурент не закрывает оба + не даёт интерактив. Целевая аудитория запроса — **главный энергетик, главный механик, инженер ОГМ/ОГЭ** — это в точности ICP из `strategy-2026.md`.

**Структурное преимущество под Яндекс:** Яндекс сильно весит **поведенческие факторы** (дочитываемость, возвраты, отсутствие pogo-sticking). Excel-шаблон (скачивание) + JS-калькулятор межремонтного цикла (вовлечение, возвраты) дают сигналы, которые blanker.ru технически повторить не может.

## 5. Спецификация JS-калькулятора межремонтного цикла

**Назначение:** дать практику быстро прикинуть, через сколько дней наступит следующий ремонт, и сколько ТО/ремонтов в год → это удерживает на странице и провоцирует возвраты (поведенческий сигнал для Яндекса).

**Поля ввода:**
| Поле | Тип | Подсказка/placeholder |
|---|---|---|
| Наработка между ремонтами одного вида, моточасов | number | напр. 4320 |
| Средняя суточная наработка, моточасов/сутки | number | напр. 16 (две смены) |
| Дата последнего ремонта | date | опционально |

**Расчёт (вывод):**
- Дней до следующего ремонта = `наработка_между_ремонтами / средняя_суточная_наработка` (округление вверх).
- Дата следующего ремонта = `дата_последнего + дней_до_следующего` (если дата задана).
- Ремонтов в год = `(365 × средняя_суточная_наработка) / наработка_между_ремонтами` (1 знак после запятой).

**UX/тех:**
- Чистый HTML+CSS+vanilla JS, без внешних библиотек (CSP в `nginx/etoir.conf` блокирует сторонние inline-скрипты — проверить, что свой inline-скрипт разрешён через `script-src 'self'` или nonce).
- Расчёт по `input` без перезагрузки. Валидация: положительные числа, деление на ноль → подсказка.
- Доступность: `<label for>` у каждого поля, `aria-live="polite"` на блоке результата.
- Цели Яндекс.Метрики: событие `ppr_calc_used` при первом изменении поля (см. `docs/SEO-TODO.md` — цели Метрики).
- Не утяжелять LCP: виджет ниже первого экрана, скрипт `defer`.

## 6. Спецификация Excel-шаблона (`grafik-ppr-shablon.xlsx`)

Путь: `landing/blog/grafik-ppr-oborudovaniya-obrazec/files/grafik-ppr-shablon.xlsx`

**Лист 1 «График ППР (год)»:**
- Столбцы: № / Наименование оборудования / Инв. номер / Цех, участок / Дата последнего ремонта / Межремонтный цикл, ч / Янв … Дек (12 столбцов) / Годовая трудоёмкость, чел-ч.
- В месячных ячейках — буквенная кодировка: **Т** (текущий), **С** (средний), **К** (капитальный), **О** (осмотр/ТО).
- Условное форматирование: цвет ячейки по типу работ (легенда на листе).
- 8–10 строк примера заполнения (станки/насосы/электродвигатели).

**Лист 2 «Расчёт цикла»:**
- Формулы: дней до ремонта = `=ОКРУГЛВВЕРХ(нараб_между / сут_нараб;0)`, ремонтов в год = `=ОКРУГЛ(365*сут_нараб/нараб_между;1)`. Должны совпадать с логикой JS-калькулятора.

**Лист 3 «Легенда и инструкция»:** расшифровка кодов, ссылка на ГОСТ 18322-2016, дисклеймер.

**Отдача файла (nginx):** `Content-Disposition: attachment`, корректный `Content-Type`, **`X-Robots-Tag: noindex`** — чтобы файл не отбирал трафик у HTML-страницы. Прописать в `nginx/etoir.conf`.

## 10. Поведенческие факторы под Яндекс (конкретные приёмы)

Яндекс ранжирует во многом по поведению. Заложить в статью:
- **Калькулятор и кнопка «Скачать»** выше «линии сгиба внимания» (после 1-го экрана) — провоцируют действие, а не отказ.
- **Оглавление с якорями** — снижает отказы, даёт переходы внутри страницы.
- **Quick-answer** в начале — закрывает интент сразу (Яндекс не любит «воду до ответа»).
- **Таблицы и буквенная кодировка** — высокая «сканируемость», дольше держит.
- **FAQ-аккордеон** в конце — увеличивает время на странице и глубину.
