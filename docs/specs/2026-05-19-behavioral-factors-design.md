## Часть B. Tasks 16–17 (поведенческие факторы)

### B1. ROI-калькулятор (Task 16) — на главной

**Файл:** `landing/index.html`, размещение — после «Реальные цифры», перед таблицей сравнения (A3).

**3 поля ввода (range-слайдеры с подписями):**

| Поле | Диапазон | Дефолт |
|---|---|---|
| Часов простоев в месяц по парку | 0–500 | 40 |
| Средняя стоимость часа простоя, ₽ | 5 000 – 500 000 | 50 000 |
| Единиц критичного оборудования | 1–500 | 30 |

**Формула расчёта (всё на клиенте, обновление мгновенно):**

```js
const ROI_CONFIG = {
  downtimeReductionPct: 0.38,     // -38% простоев (из hero)
  sparesAvgCostPerUnit: 80000,    // ₽/единицу/год — среднее по бенчмаркам
  sparesReductionPct: 0.05,       // 5% экономия запчастей
  laborHourlyRate: 1500,          // ₽/час слесаря
  laborWasteRatio: 0.25,          // 25% непродуктивного времени
  etoirAnnualCost: 1080000        // 90 000 ₽/мес × 12
};

downtime_savings = hours * 12 * costPerHour * 0.38;
spares_savings   = units * 80000 * 0.05;
labor_savings    = hours * 12 * 1500 * 0.25;
total            = downtime_savings + spares_savings + labor_savings;
payback_months   = (1080000 / total) * 12;
```

**Вывод:**
- Большая цифра «total ₽ в год» (форматирование `Intl.NumberFormat('ru-RU')`)
- 3 строки разбивки: «− N ₽ на простоях», «− N ₽ на запчастях», «− N ₽ на трудозатратах»
- Строка «Окупаемость эТОИР: ~N месяцев»
- Кнопка «Узнать точные цифры для вашего завода» → якорь `#contact`
- Микро-ссылка «откуда эти цифры?» → раскрывает аккордеон B3

**Событие в Я.Метрику и GA4:** `roi_calculated` при изменении любого из слайдеров (с debounce 500 мс) — для измерения вовлечённости.

### B4. Sticky-оглавление в статьях (Task 17)

**Файлы:** все три `landing/blog/<slug>/index.html`.

**Разметка:** `<article>` оборачивается в `<div class="article-layout">` с CSS-grid:
- Desktop (`≥1024px`): `grid-template-columns: 280px 1fr`, `gap: 48px`. Sidebar `<aside class="article-toc">` — `position: sticky; top: 100px; max-height: calc(100vh - 120px); overflow-y: auto`.
- Mobile (`<1024px`): `grid-template-columns: 1fr`. Тот же контейнер преобразуется в `<details class="article-toc-mobile">` непосредственно после первого `<p>` статьи; `<summary>` — «📋 Содержание (N разделов)».

**Поведение:**
- TOC генерируется JS-ом из всех `<h2>` статьи. Если `<h2>` нет `id`, добавляется автоматически из slugify(text).
- Scroll-spy через IntersectionObserver: текущий H2 (тот, что ближе всего к верху viewport, в зоне 0–30% высоты) получает в TOC `class="active"`.
- Клик по пункту TOC: плавный скролл (`scroll-behavior: smooth`) и обновление URL-хеша.

**Реализация:** один inline-скрипт `<script>` в каждой статье. Логика идентична — копипаста допустима (3 статьи). При появлении 4-й — выносим в `landing/blog/_toc.js`.

### B5. Sticky mobile CTA (Task 17)

**Файлы:** `landing/index.html` + 3 статьи блога.

**Появление:**
- Только при ширине `<768px`.
- На главной — после скролла мимо hero (через IntersectionObserver на маркере в конце hero-секции).
- В статьях — после скролла >40% высоты статьи.

**Разметка:**
```html
<div class="sticky-cta-mobile" aria-hidden="true">
  <span class="sticky-cta-text">Бесплатное демо за 30 минут</span>
  <a href="#contact" class="sticky-cta-btn">Получить демо</a>
  <button class="sticky-cta-close" aria-label="Закрыть">✕</button>
</div>
```

**Стили:** `position: fixed; bottom: 0; left: 0; right: 0; height: 56px;` — primary blue фон, золотая кнопка справа. По умолчанию `transform: translateY(100%)`; класс `.visible` поднимает.

**Закрытие:** клик по `✕` пишет ключ `sticky-cta-dismissed` в `sessionStorage`. При наличии ключа — плашка не показывается до конца сессии.

**Учёт CTA:** клик по `.sticky-cta-btn` шлёт событие `sticky_cta_click` в Я.Метрику и GA4.
