## ВАЖНО: язык ядра и почему

Аудитория ниши в РФ — российские заводы, ремонтные службы, главные инженеры, мастера, слесари. По нашему анализу выдачи Яндекса они **массово ищут на русском**, а не английскими аббревиатурами:

| Что ищут редко (англ., профи-форма) | Что ищут массово (русск., рабочая форма) |
|---|---|
| `CMMS`, `EAM`, `CMMS система` | `программа для ТОиР`, `система учёта ремонтов`, `автоматизация ТОиР` |
| `MTBF`, `MTTR` | `наработка на отказ`, `коэффициент готовности оборудования` |
| `work order`, `naryad-zakaz` транслитом | `наряд-заказ на ремонт оборудования`, `заказ-наряд образец` |
| `preventive maintenance` | `планово-предупредительный ремонт`, `график ППР` |
| `asset registry` | `паспорт оборудования`, `журнал учёта оборудования` |
| `defect report` | `дефектная ведомость`, `акт дефектовки` |

Англоязычные термины оставляем **только как вторичные ключи** внутри статей (для длинного хвоста и для аудитории топ-менеджмента), но title/H1/первый абзац — **только русские формы**.

## Апгрейд существующих статей (обязательно до деплоя)

Все три статьи уже имеют корректные мета-данные, Schema, осмысленный текст. **Технические пробелы** — те же у всех, плюс есть точечные.

### Общие фиксы — в каждую из 3 статей (`landing/blog/<slug>/index.html`)

1. **Заменить битый CTA-якорь.** Сейчас 3–4 кнопки «Получить демо» имеют `href="#contact-form"` — это якорь текущей страницы, форма же на главной. Заменить на `href="/#contact-form"`.
2. **Добавить мета-теги, которых нет:**
   ```html
   <meta name="robots" content="index, follow, max-image-preview:large">
   <meta name="twitter:card" content="summary_large_image">
   <meta name="twitter:title" content="<совпадает с og:title>">
   <meta name="twitter:description" content="<совпадает с og:description>">
   <meta name="twitter:image" content="<URL уникального hero статьи>">
   ```
3. **Добавить картинки.** В подпапке `landing/blog/<slug>/img/` — минимум 4 файла: hero (1200×630) + 2–3 иллюстрации/схемы по тексту, каждая в WebP + PNG, ≤ 200 КБ. См. блок «Минимальный набор картинок» ниже.
4. **Подменить `og:image` на уникальный hero статьи** (вместо общей `/images/og-image.png`).
5. **Preload hero** в `<head>`:
   ```html
   <link rel="preload" as="image" type="image/webp"
         href="/blog/<slug>/img/hero.webp" fetchpriority="high">
   ```
6. **Перелинковка** — добавить блок «Читайте также» в конце + врезные ссылки по тексту:
   - ≥ 3 ссылки на главную и её якоря: `/`, `/#features-section`, `/#industries`, `/#roi-calculator`, `/#contact-form`
   - ≥ 1 ссылку на каждую из других двух статей блога
7. **Обновить `dateModified`** в `Article` JSON-LD при выкатке апгрейда — на дату фактического обновления.

## Общие правила для всех статей (обязательны)

- **URL:** `/blog/<slug-русское-в-латинице>/`, без `index.html` в URL.
- **HTML-шаблон:** наследуем разметку существующих `landing/blog/*/index.html`, тот же `styles.css`, тот же sprite иконок.
- **Schema.org:** `Article` + `BreadcrumbList`; если есть FAQ-блок — `FAQPage`; если есть таблица сравнения — `Table` через JSON-LD `Dataset`.
- **Open Graph / Twitter Card:** обязательно, `og:image` 1200×630 (WebP + PNG fallback).
- **Плотность главного ключа:** 1.5–2.5 %. Точное вхождение в `<title>`, `<h1>`, в первых 100 словах, в одном `<h2>`, в `alt` главной картинки, в `<strong>` 1–2 раза.
- **Перелинковка:** ≥ 3 ссылки на главную (`/`, `/#features-section`, `/#industries`, `/#pricing`), ≥ 2 ссылки на другие статьи блога.
- **CTA-блок:** inline-CTA после 1500–2000 знаков + финальный CTA с якорем на `/#contact-form` («Получить демо эТОИР»).
- **Картинки:** WebP + PNG fallback, ширина 1200 px, ≤ 200 КБ, осмысленный alt с ключом, `loading="lazy"` для всех кроме первой.
- **Дата:** `datePublished` + `dateModified` в JSON-LD; человекочитаемая дата «Обновлено: <месяц год>» в шапке.
- **Якоря (`id`)** у каждого H2 — для оглавления и Яндекс «быстрых ответов».
- **Sitemap + RSS:** добавить URL в `landing/sitemap.xml` (с `<lastmod>`) и в `landing/rss.xml`.
- **Lighthouse mobile Performance ≥ 90** обязательно перед публикацией.
- **Яндекс.Вебмастер:** отправить URL на переобход; через 7 дней — снять позиции и индексацию.
